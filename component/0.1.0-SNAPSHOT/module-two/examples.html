Examples

Examples are files used to explain the usage of some code or functionality.

Properties

Properties are declared in maven using <properties> tag like the following example.

Properties
	<properties>
		<java.version>17</java.version> (1)
		
		<!-- Plugins -->
		<git-commit-id-maven-plugin.version>6.0.0</git-commit-id-maven-plugin.version> (2)
	</properties>
1 Java version
2 Git Commit Plugin version to generate git.properties file

Build Info

It is needed to add the actuator spring dependency and add some configuration in the application properties file to expose info endpoints.

Maven

Maven information are extracted by using a configuring Spring maven plugin with the following configuration.

Properties
			<plugin>
				<groupId>org.springframework.boot</groupId>
				<artifactId>spring-boot-maven-plugin</artifactId>
				<executions>
					<execution>
						<goals>
							<!-- Create metadata needed to expose build info at /actuator/info -->
							<!-- Check target/classes/META-INF/build-info.properties -->
							<goal>build-info</goal> (1)
						</goals>
					</execution>
				</executions>
				<configuration>
					<!-- Enable layering when packaging the jar file for performance -->
					<layers>
						<enabled>true</enabled>
					</layers>
					<!-- Add additional build info properties in metadata -->
					<additionalProperties>  (2)
						<custom.key1>value1</custom.key1>
						<custom.key2>value2</custom.key2>
						<!-- Use properties from maven to expose dependencies or configuration -->
						<plugins.git-commit>${git-commit-id-maven-plugin.version}</plugins.git-commit>
					</additionalProperties>  (3)
					<!-- To exclude properties added into build info -->
					<excludeInfoProperties>
						<infoProperty>custom.key1</infoProperty>
					</excludeInfoProperties>
				</configuration>
			</plugin>
1 Add a goal to create build info
2 Include additional properties
3 Exclude any properties for security reasons.

Git

Git information are extracted by using a maven plugin with the following configuration.

Properties
			<plugin>
				<groupId>io.github.git-commit-id</groupId> (1)
				<artifactId>git-commit-id-maven-plugin</artifactId> (2)
				<version>${git-commit-id-maven-plugin.version}</version> (3)
				<configuration>
					<generateGitPropertiesFile>true</generateGitPropertiesFile> (4)
					<commitIdGenerationMode>full</commitIdGenerationMode>
					<includeOnlyProperties> (5)
						<includeOnlyProperty>git.commit.id.abbrev</includeOnlyProperty>
						<includeOnlyProperty>git.commit.id.full</includeOnlyProperty>
						<includeOnlyProperty>git.commit.message.short</includeOnlyProperty>
						<includeOnlyProperty>git.commit.time</includeOnlyProperty>
					</includeOnlyProperties>
				</configuration>
			</plugin>
1 Group Id (Changed from v5.x).
2 Artifact Id.
3 Artifact Version from properties.
4 Enabled by default to generate git.properties file
5 To Include only specific metadata from Git for security reasons.
